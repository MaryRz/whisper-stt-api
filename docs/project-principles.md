# Project Principles

This project is developed according to the following engineering principles.

## Clean Architecture

Business logic should remain independent from HTTP and infrastructure.

---

## Separation of Concerns

Each module should have a single responsibility.

---

## Security First

User input is never trusted without validation.

---

## Readability

Code should be understandable without additional explanation.

---

## Maintainability

The project should remain easy to modify after months of development.

---

## Testability

Every important component should be independently testable.

---

## Production Mindset

Every design decision should consider scalability, deployment, and future maintenance.
